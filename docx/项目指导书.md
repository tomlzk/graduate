课题名称: 基于WEB的高校学生考证服务系统设计

设计（论文）主要内容
针对大学生在各类考证中面临的疑问与困难，本课题设计一个基于WEB的高校学生考证服务系统，主要包含考公、考研、四级考试、六级考试、雅思考试、托福考试、计算机等级考试等模块，用户能实现注册、登录、修改密码、发帖、删除或修改自己所发的帖子、提问、查看信息、给其他用户回答问题等功能，管理员能实现对用户的名称与密码的管理，对帖子的增加、删除与修改管理，能发布系统公共信息（例如，年度四六级考试时间安排等），要求界面友好，使用方便。


技术栈：前端用vue3
后端用spring boot
数据库用mysql

---

## 一、系统需求分析

### 1.1 项目背景

随着社会竞争日益激烈，越来越多的高校学生选择通过考取各类证书来提升自身竞争力。然而，考证过程中学生往往面临信息分散、经验难以获取、缺乏交流平台等问题。目前市面上缺少一个专门面向高校学生、覆盖多种考证类型的综合服务平台。因此，开发一个基于WEB的高校学生考证服务系统具有重要的实际意义。

### 1.2 用户角色分析

本系统涉及两类用户角色：

| 角色 | 描述 | 权限概述 |
|------|------|----------|
| **普通用户（学生）** | 系统的主要使用者，高校在校学生 | 注册/登录、个人信息管理、浏览考证信息、发帖交流、提问与回答 |
| **管理员** | 系统的管理维护人员 | 用户管理、帖子管理、公告信息发布与管理、考试模块管理 |

### 1.3 功能性需求

#### 1.3.1 普通用户功能需求

| 编号 | 功能模块 | 功能描述 |
|------|----------|----------|
| U-01 | 用户注册 | 用户填写用户名、密码、邮箱等信息完成注册 |
| U-02 | 用户登录 | 用户通过用户名和密码登录系统 |
| U-03 | 修改密码 | 用户在登录状态下可修改个人密码 |
| U-04 | 个人信息管理 | 用户查看和编辑个人资料（昵称、头像、简介等） |
| U-05 | 浏览考证模块 | 用户可浏览考公、考研、四级、六级、雅思、托福、计算机等级考试等模块 |
| U-06 | 查看考试信息 | 用户查看各考试模块下的考试时间、报名方式、备考资料等信息 |
| U-07 | 发布帖子 | 用户在对应考证模块下发布经验分享、资料分享等帖子 |
| U-08 | 修改帖子 | 用户可修改自己已发布的帖子内容 |
| U-09 | 删除帖子 | 用户可删除自己已发布的帖子 |
| U-10 | 提问功能 | 用户可在各考证模块下发布问题 |
| U-11 | 回答问题 | 用户可对其他用户发布的问题进行回答 |
| U-12 | 查看公告 | 用户可查看管理员发布的系统公告信息 |

#### 1.3.2 管理员功能需求

| 编号 | 功能模块 | 功能描述 |
|------|----------|----------|
| A-01 | 管理员登录 | 管理员通过专用账号密码登录后台管理系统 |
| A-02 | 用户管理 | 查看用户列表，修改用户名称，重置用户密码，禁用/启用用户 |
| A-03 | 帖子管理 | 查看所有帖子，新增、修改、删除任意帖子 |
| A-04 | 问答管理 | 查看所有问答内容，删除违规问题或回答 |
| A-05 | 公告管理 | 发布、修改、删除系统公告信息（如考试时间安排、报名通知等） |
| A-06 | 考试模块管理 | 管理各考试模块的基本信息（考试简介、考试时间、报名链接等） |

### 1.4 非功能性需求

| 需求类型 | 描述 |
|----------|------|
| **易用性** | 界面设计简洁直观，操作便捷，符合大学生使用习惯 |
| **安全性** | 用户密码加密存储，接口进行权限校验，防止SQL注入和XSS攻击 |
| **响应性** | 页面加载时间不超过3秒，支持主流浏览器（Chrome、Edge、Firefox） |
| **可维护性** | 代码结构清晰，前后端分离，便于后期扩展和维护 |
| **兼容性** | 支持PC端主流浏览器访问，页面做基本的响应式适配 |

## 二、考证模块详细说明

系统包含以下七大考证模块，每个模块均提供信息浏览、帖子交流、问答互动功能：

| 模块编号 | 模块名称 | 模块内容说明 |
|----------|----------|-------------|
| M-01 | 考公模块 | 国家公务员考试、省考等公务员考试相关信息，包含行测、申论备考经验，岗位选择指南等 |
| M-02 | 考研模块 | 全国硕士研究生考试相关信息，包含初试/复试经验、院校选择、专业课备考资料等 |
| M-03 | 四级考试模块 | 大学英语四级考试（CET-4）相关信息，包含听力、阅读、写作、翻译备考技巧等 |
| M-04 | 六级考试模块 | 大学英语六级考试（CET-6）相关信息，包含备考策略、真题解析、高分经验等 |
| M-05 | 雅思考试模块 | 雅思（IELTS）考试相关信息，包含听说读写各科备考方法、考试报名流程等 |
| M-06 | 托福考试模块 | 托福（TOEFL）考试相关信息，包含备考资源、考试技巧、成绩要求等 |
| M-07 | 计算机等级考试模块 | 全国计算机等级考试（NCRE）相关信息，包含一级至四级各科目备考资料等 |

## 三、系统架构与技术选型

### 3.1 整体架构

本系统采用前后端分离的B/S架构：

```
┌─────────────┐     HTTP/REST API     ┌──────────────┐     JDBC      ┌─────────┐
│   前端 Vue3  │  ◄──────────────────►  │  Spring Boot │  ◄──────────►  │  MySQL  │
│  (浏览器端)  │                        │   (服务端)    │               │ (数据库) │
└─────────────┘                        └──────────────┘               └─────────┘
```

### 3.2 技术选型明细

| 层次 | 技术 | 版本建议 | 说明 |
|------|------|----------|------|
| 前端框架 | Vue 3 | 3.x | 组合式API，响应式开发 |
| 前端构建 | Vite | 5.x | 快速开发构建工具 |
| 前端UI库 | Element Plus | 2.x | Vue3组件库，提供丰富UI组件 |
| 前端路由 | Vue Router | 4.x | 单页面应用路由管理 |
| 前端状态管理 | Pinia | 2.x | Vue3推荐状态管理方案 |
| 前端HTTP | Axios | 1.x | HTTP请求库 |
| 后端框架 | Spring Boot | 3.x | Java后端开发框架 |
| ORM框架 | MyBatis-Plus | 3.5.x | 简化数据库操作 |
| 数据库 | MySQL | 8.0 | 关系型数据库 |
| 接口规范 | RESTful API | - | 前后端数据交互规范 |
| 权限认证 | JWT | - | 基于Token的用户认证方案 |

## 四、数据库设计概要

### 4.1 核心数据表

| 表名 | 中文名 | 主要字段 | 说明 |
|------|--------|----------|------|
| `user` | 用户表 | id, username, password, email, nickname, avatar, role, status, create_time | 存储用户基本信息和角色 |
| `exam_module` | 考试模块表 | id, name, description, icon, sort_order, create_time | 七大考证模块基础信息 |
| `post` | 帖子表 | id, user_id, module_id, title, content, view_count, create_time, update_time | 用户发布的帖子内容 |
| `question` | 问题表 | id, user_id, module_id, title, content, status, create_time | 用户发布的提问 |
| `answer` | 回答表 | id, question_id, user_id, content, create_time | 用户对问题的回答 |
| `announcement` | 公告表 | id, admin_id, title, content, is_top, create_time, update_time | 管理员发布的系统公告 |
| `exam_info` | 考试信息表 | id, module_id, title, content, exam_time, register_url, create_time | 各模块下具体考试信息 |

### 4.2 表关系说明

- `user` 1:N `post`（一个用户可发多个帖子）
- `user` 1:N `question`（一个用户可发多个问题）
- `user` 1:N `answer`（一个用户可回答多个问题）
- `exam_module` 1:N `post`（一个模块下有多个帖子）
- `exam_module` 1:N `question`（一个模块下有多个问题）
- `exam_module` 1:N `exam_info`（一个模块下有多条考试信息）
- `question` 1:N `answer`（一个问题有多个回答）
